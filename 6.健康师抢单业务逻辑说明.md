

### 二、健康师抢单业务逻辑说明

#### 1. 核心概念
**抢单功能**：指平台发布一个未指定服务提供者的标准化服务包，允许符合资质的健康师主动申请承接该订单的机制。

#### 2. 参与角色与前提条件

| 角色 | 前提条件 |
| :--- | :--- |
| **平台运营** | - 有权创建和发布平台服务包<br>- 可配置抢单规则（如健康师等级要求） |
| **健康师** | - 账号通过资质审核<br>- 健康师等级达到服务包设定的要求（如：金牌）<br>- 擅长领域与服务包的健康问题分类匹配 |
| **C端用户** | - 购买的是**平台服务包**，而非指定健康师的服务 |

#### 3. 业务流程步骤

1.  **平台发布服务包**
    *   运营人员在M17运营后台创建标准化服务包（如：“21天减脂训练营”）。
    *   配置抢单规则：设定可抢单的健康师最低等级、服务包价格、服务标准、有效期等。

2.  **C端用户购买服务包**
    *   用户在服务市场（M04）看到并购买该平台服务包，支付成功后生成订单，订单状态为`待分配健康师`。

3.  **系统推送抢单机会**
    *   系统根据服务包的**健康问题分类**和**抢单规则**，自动将抢单机会推送至符合条件的健康师的B端工作台（M03.1）的“抢单中心”或任务列表。

4.  **健康师抢单**
    *   健康师在抢单列表看到订单概要（隐藏用户敏感信息）。
    *   健康师点击“抢单”申请承接。
    *   系统校验：实时校验该健康师是否仍符合条件（等级、擅长领域、无违规记录）。

5.  **订单分配与确认**
    *   **分配模式1：先到先得**：第一个成功抢单的健康师直接获得订单。
    *   **分配模式2：平台指派**：多名健康师抢单后，由平台运营根据综合因素（评分、响应速度等）手动指派最优健康师。
    *   订单状态更新为`已分配健康师`，并通知C端用户。

6.  **服务开始**
    *   健康师与用户建立服务关系，开始履约。

#### 4. 规则与风控要点

*   **等级门槛**：由平台运营动态调整，例如初期可设为“银牌及以上”，后期可提至“金牌”。
*   **频次限制**：同一健康师单位时间内抢单数量上限，防止垄断。
*   **超时机制**：若一段时间内无人抢单，系统自动提醒更多健康师或由平台运营手动分配。
*   **违规处理**：健康师抢单后无故不服务或服务质量差，将影响其抢单权限和等级评分。
