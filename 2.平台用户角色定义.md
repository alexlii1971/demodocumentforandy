### 平台用户角色定义 (V3.1)

基于V3.1版本的功能更新，对平台用户角色定义进行相应调整和补充：

| 角色 | 核心诉求 | 权限范围与功能入口 | V3.1更新说明 |
|------|----------|-------------------|-------------|
| **C端用户 (Customer)** | 获得专业、可信、有效的健康服务与产品，解决健康问题。 | 入口：C端App/小程序。功能：消费服务、购买商品、参与社区、管理健康数据、使用积分。所有操作入口集中于C端个人中心(M02)。 | **【增强】** 新增健康档案AI评估、每日行动指引、动态流互动等功能 |
| **健康师 (Health Coach)** | 通过专业知识变现，建立个人品牌，获取客户。其权限受限于所购买的平台服务套餐。 | 入口：B端工作台-健康师版(M03.1)。功能：发布咨询/课程/直播/社群/文章（依套餐而定）、使用AI辅助工具（依套餐而定）、管理客户、查看数据。**新增**：参与平台服务包抢单、人工积分奖励、动态流内容发布。 | **【重大更新】** 新增抢单功能、人工积分奖励、动态流内容发布权限 |
| **卖家 (Merchant)** | 销售健康相关的实体商品（器械、食品、补剂）。 | 入口：B端工作台-卖家版(M03.3)。功能：发布和管理商品、处理订单与售后、参与营销活动。无法：发布任何咨询服务。 | **【保持稳定】** 功能无重大变化 |
| **保险公司 (Insurer)** | 销售健康险；通过平台数据降低保赔付率。 | 入口：B端工作台-保险版(M03.4)。功能：发布和管理保险产品、查看保单、处理理赔接口对接。 | **【保持稳定】** 功能无重大变化 |
| **平台服务运营 (Platform Service Operator)** | 管理并销售平台的服务套餐产品（基础版/直播版/全能版），处理B端用户的套餐购买、续费、升级。 | 入口：平台运营后台(M17)中的专属模块。功能：配置和上架不同的服务套餐产品、管理B端套餐订单。**新增**：抢单规则配置、动态流内容管理。权限隔离：仅能操作与套餐商品相关功能，无法访问内容审核、用户治理等核心业务功能。 | **【扩展】** 新增抢单规则配置和动态流内容管理权限 |
| **平台内容运营 (Platform Content Operator)** | 【细分角色】负责健康社区(M11)、问答平台(M12)的内容审核与版主管理，维护社区氛围。 | 入口：平台运营后台(M17)。功能：内容审核后台、举报处理、版主任命。**新增**：动态流内容审核与管理。权限隔离：仅限UGC内容管理相关功能。 | **【扩展】** 新增动态流内容审核与管理权限 |
| **平台交易运营 (Platform Transaction Operator)** | 【细分角色】负责健康服务(M04)、电商(M09)、直播(M07)等交易生态的监控、促销活动创建及纠纷处理。 | 入口：平台运营后台(M17)。功能：交易监控、活动运营、优惠券发放、投诉仲裁。**新增**：抢单订单监控、抢单纠纷处理。权限隔离：仅限交易与营销相关功能。 | **【扩展】** 新增抢单业务监控和纠纷处理权限 |
| **平台超级管理员 (Platform Super Admin)** | 【细分角色】拥有所有业务运营权限，制定全局规则，管理所有业务运营角色账号，是运营层面的最高负责人。 | 入口：平台运营后台(M17)。功能：全平台功能开关、全局规则配置、运营账号权限分配、查看全平台核心数据。**新增**：抢单系统全局配置、动态流系统全局管理。 | **【扩展】** 新增抢单系统和动态流系统的全局管理权限 |
| **系统管理员 (System Admin)** | 系统维护、技术支撑、基础架构管理。 | 入口：系统后台。功能：服务器、数据库、网络安全、备份恢复。权限隔离：不参与任何业务运营，与业务后台物理隔离。 | **【保持稳定】** 功能无重大变化 |
| **数据买家 (Data Buyer)** | 购买匿名化、聚合化的数据洞察报告或API接口。 | 入口：数据市场门户网站。功能：浏览、购买、下载数据产品。 | **【保持稳定】** 功能无重大变化 |
| **投资人 (Investor)** | 通过只读API获取平台脱敏核心数据，进行投后管理。 | 入口：通过API。权限：仅能通过API访问预设的全局经营看板数据。 | **【保持稳定】** 功能无重大变化 |
| **AI赋能代理 (AI Empowerment Agent)** |  |  |  |
| *定位：本平台核心的虚拟赋能系统，并非一个独立的用户角色。它不具备账户，不参与分成，不承担服务责任。* |  |  |  |
| *功能：作为底层能力深度集成至各业务模块（如M04, M07, M11, M12, M16, M18, M19, M20），通过算法为平台运营提供管理工具，为B端用户（健康师等）提供内容创作、客户服务、数据分析等方面的智能辅助，旨在提升效率与体验。* |  |  | **【扩展】** 新增支持M19动态流系统和M20抢单系统的AI能力 |
| *责任关系：所有由AI生成的内容或建议，必须经过健康师或平台运营人员的审核、修改与确认后方可生效，最终责任由操作该功能的人类用户承担。* |  |  |  |

### V3.1版本角色权限主要变化总结：

1. **健康师角色重大扩展**：
   - 新增平台服务包抢单权限
   - 新增人工积分奖励功能
   - 增强动态流内容发布和互动权限

2. **平台运营角色权限细化**：
   - 平台服务运营：新增抢单规则配置权限
   - 平台内容运营：新增动态流内容管理权限
   - 平台交易运营：新增抢单业务监控权限
   - 平台超级管理员：新增抢单系统和动态流系统全局管理权限

3. **AI赋能代理角色扩展**：
   - 新增对M19动态流系统和M20抢单系统的AI能力支持

4. **其他角色保持稳定**：
   - C端用户、卖家、保险公司、系统管理员、数据买家、投资人角色功能无重大变化，主要是体验优化和功能增强
