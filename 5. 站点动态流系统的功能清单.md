### 三、站点动态流系统的功能清单

#### 系统名称：M19 - 站点动态流系统 (Platform Activity Feed System)

**核心目标**：聚合全平台用户与健康师的关键活动，生成一个统一、活跃、可个性化订阅的信息流，增强社区氛围和用户粘性。

---

#### 1. 动态内容生产模块

| 功能点 | 触发来源/角色 | 动态内容示例（模板化） | 数据来源模块 |
| :--- | :--- | :--- | :--- |
| **用户健康行动** | C端用户 | - “[用户昵称] 完成了今日打卡 [行动名称]”<br>- “[用户昵称] 体重减轻了 1公斤”<br>- “[用户昵称] 今日步行 10,000步” | M02 (个人中心) |
| **用户成就获取** | 系统自动 | - “[用户昵称] 获得了 [勋章名称] 勋章”<br>- “[用户昵称] 连续打卡 7天” | M14 (积分系统) |
| **社区内容发布** | C端/健康师 | - “[用户/健康师昵称] 发布了新问题：[问题标题]”<br>- “[健康师昵称] 发表了一篇文章：[文章标题]” | M11 (健康社区)<br>M12 (问答平台) |
| **健康师服务动态** | 健康师 | - “[健康师昵称] 发布了一个新课程：[课程名]”<br>- “[健康师昵称] 即将开始直播：[直播主题]” | M06 (知识商城)<br>M07 (直播中心) |
| **平台运营活动** | 平台运营 | - “平台新活动：[活动名称] 已上线，快来参与！”<br>- “系统公告：[政策更新摘要]” | M17 (运营后台) |
| **关系互动动态** | 系统自动 | - “[健康师昵称] 点赞了你的动态”<br>- “[用户昵称] 关注了你” | 各互动模块 |

---

#### 2. 动态流消费与交互模块

| 功能点 | 描述 | 前端位置 |
| :--- | :--- | :--- |
| **全局动态流首页** | 聚合所有公开动态，按时间倒序排列，作为平台首页或独立“发现”页。 | C端App/网站首页 |
| **关注流** | 仅显示用户已关注的健康师/其他用户的动态。 | C端个人中心/专属Tab |
| **个性化推荐流** | 基于用户健康标签、历史行为，由算法推荐相关动态（如：同领域健康师动态）。 | 混合在全局流中或独立区块 |
| **动态交互** | 支持对动态进行点赞、评论、分享（分享至站外或站内社群）。 | 每条动态下方 |
| **动态过滤与筛选** | 用户可按动态类型（如：只看“用户行动”、只看“健康师文章”）筛选内容。 | 动态流顶部筛选器 |
| **@提及通知** | 用户在发帖、评论时@他人，被@的用户会在消息中心收到通知，并可能出现在其动态流中。 | 集成于发布与通知系统 |

---

#### 3. 动态管理后台 (位于M17运营后台)

| 功能点 | 描述 | 权限角色 |
| :--- | :--- | :--- |
| **动态内容审核** | 敏感或疑似违规动态进入审核队列，运营人员可进行删除或忽略操作。 | 平台内容运营 |
| **动态类型管理** | 配置哪些类型的行为可以触发生成动态（功能开关）。 | 平台超级管理员 |
| **动态模板管理** | 编辑各类动态的显示文案模板。 | 平台超级管理员 |
| **动态推送管理** | 运营可将重要动态（如官方活动）手动置顶或强推送。 | 平台业务运营 |
| **数据看板** | 查看动态流相关数据：每日动态发布量、互动率、热门动态等。 | 平台业务运营 |

---

#### 4. 技术实现要点

| 方面 | 说明 |
| :--- | :--- |
| **数据源监听** | 通过消息队列（如RabbitMQ/Kafka）监听各业务模块（M02, M06, M11等）的事件，异步生成动态。 |
| **动态存储** | 使用独立的`activities`表存储动态内容，包含：用户ID、动态类型、目标对象ID、模板数据、时间戳等。 |
| **Feed生成** | 用户访问动态流时，实时查询或为其生成预计算的动态列表（关注流需合并多人的动态）。 |
| **性能优化** | 对热门动态流进行分页、缓存，避免大规模连表查询影响性能。 |

---

**总结**：此动态流系统作为平台的“社交脉搏”，通过自动化采集和展示用户与健康师的各类活动，有效提升平台的活跃度、用户粘性和社区归属感。
